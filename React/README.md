# Design Systems & Auditing an Existing App (Codebar Event)

## Shows how design systems help create consistent and maintainable user interfaces.

### Overview

This project was created during a Codebar workshop, focusing on integrating a design system into an existing React application.

The goal was to understand how reusable components and design systems improve consistency, accessibility, and maintainability across an application.

---

### Tech Stack

- React
- TypeScript
- Node.js
- Vite
- [GOV.UK Design System](https://design-system.service.gov.uk/)
- govuk-frontend package (^6.1.0)

---

### Features

- Integration of [GOV.UK design system components](https://design-system.service.gov.uk/components/)
- Use of reusable UI components
- Improved UI consistency across the application
- Exploration of component-driven development

---

### Getting Started

#### Run Locally

**1. Clone the repository:**

```bash
git clone https://github.com/devkristie/codebar-design-systems-audit-workshop-app.git
cd codebar-design-systems-audit-workshop-app
```

**2. Install dependencies:**

```bash
npm install
```

**3. Run the development server:**

```bash
npm run dev
```

The app will be running at http://localhost:5173.

---

### Prerequisites

- Node.js (v20 or higher recommended)
- npm (comes with Node.js)
